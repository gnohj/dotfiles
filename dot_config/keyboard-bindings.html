⌨️ Keyboard Bindings

Cross-layer reference for ctrl / alt / cmd / hyper across kanata → skhd → terminal → tmux → nvim. Manual snapshot from the chezmoi source (dot_config/keyboard-bindings.html). Update alongside skhdrc / tmux/*.conf / ghostty/config / kitty.conf.

How a keypress is resolved (precedence)

Each layer sees the key before the one below and can consume it. This is what makes a binding "global" vs "app-specific".

1kanataRemaps physical keys → homerow mods & layers. The only layer that knows left vs right ctrl.
2skhd globalmacOS event tap. Grabs rctrl/lalt/cmd/hyper combos in any app. Can tell rctrl from lctrl.
3terminal ghostty / kittyKeybinds fire before byte-encoding. Side-blind (both ctrls). Only when a terminal is focused.
4tmux root / prefixRoot binds (bind -n) fire on any key; prefix binds need C-a first. Only inside tmux.
5nvim / shell pane appWhatever the focused pane program does with the key that survived every layer above.
Remote (Tailscale SSH → VPS tmux): only layers 3–5 travel over SSH (they're bytes in the PTY). Layers 1–2 (kanata, skhd) are macOS-local and drive nothing remote. That's why single-chord nav was migrated off skhd into terminal/tmux binds.

What the modifiers physically are (kanata)

The MacBook keyboard has no dedicated modifier keys for these — they're homerow / thumb holds. Tap = the letter, hold = the modifier.

Migration & gotchas

Remote-safe (survive Tailscale SSH → VPS tmux): everything under terminal and tmux root / prefix. They're PTY bytes.

Local-only: everything under global (skhd) and aerospace — macOS event layer, drives nothing remote.

Still on skhd, pending migration: ctrl+1…9/0 (the "ctrl+digit wall" — digits come from kanata's ly1 layer) and rctrl+i (kept so nvim keeps ctrl+i).

Chord-trigger rule: never route a binding through a kanata key whose hold is a big modifier-combo (/=hyper, tab, caps/grave) — it leaks a stray modifier → macOS beep. Use plain keys (that's why popup is ctrl+. not ctrl+/).

Nesting: don't ssh from inside a local tmux — the outer tmux eats C-a. Open a bare ghostty window → ssh → remote tmux.
