# Global OpenCode Instructions

## Operations

- Only grant "auto execute" on read-only type instructions. For example,
  "Yes" to "git status". "No" to "git commit". "Yes" to "ls". "No" to "rm".

## Task Management

- When creating task documentation, check if tasks/{git branch}.md exists- you will do it a git command to get the current branch name.
- If {git branch}.md exists, append the new task documentation to that file.
- Always create task files, {git branch}.md in the tasks/ directory at the project root

## Second Brain (Obsidian Vault)

You have access to a personal knowledge base at `~/Obsidian/second-brain/`.
Read `~/Obsidian/second-brain/CLAUDE.md` for navigation instructions before
searching the vault. Use it for context when the user references past work,
patterns, cheat sheets, or asks "how did I do X before?"
